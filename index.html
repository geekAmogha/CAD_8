Pipeline Overview

On each commit: Jenkins checks out code, validates HTML/CSS, and deploys the updated site to the EC2 web server. Deployment is zero-downtime using a symlink swap strategy (placeholder). Secrets are injected via Jenkins credentials.

  1. Commit: Push changes to main or open a PR.
  2. Build: Jenkins runs lint/validation (HTML tidy, CSS checks).
  3. Package: Artifact archived for traceability.
  4. Deploy: Secure copy to EC2 and activate new release.
  5. Verify: Simple health check request.
